Onboard-CLI is a local-first, zero-telemetry engine. It runs entirely on your machine, parsing your codebase into a localized SQLite graph without sending a single byte of your proprietary code to the cloud.

## Installation

Choose the distribution method that matches your ecosystem. Both methods install the identical high-performance Go binary globally on your system.

### Option 1: Standalone Binary (Mac/Linux)
The fastest way for Unix systems engineers to get started.

```bash
curl -sL https://raw.githubusercontent.com/animesh-94/Onboard-CLI/main/install.sh | bash
```

### Option 2: PowerShell (Windows)
For Windows environments, use the native PowerShell installer.

```powershell
Invoke-WebRequest -Uri "https://raw.githubusercontent.com/animesh-94/Onboard-CLI/main/install.ps1" -OutFile "install.ps1"; .\install.ps1
```

### Option 3: NPM Package
Ideal if you are already working within a Node.js monorepo environment.

```bash
npm install -g onboard-cli
```

## Your First Trace
Once installed, navigate to the root directory of any complex repository (C++, Java, Go, or TS) and initialize the local AST cache.

```bash
cd my-massive-monorepo
onboard init
```

This will trigger the Tree-sitter engine to index your files. Once complete, launch the interactive visualizer targeting your core entry point:

```bash
onboard map --target src/main.cpp --radius 3
```

A localized web server will instantly boot at `http://localhost:3000`, rendering a high-fidelity React canvas of your execution flow.
