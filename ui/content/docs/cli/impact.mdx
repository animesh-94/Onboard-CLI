Never break production blindly. Standard IDE tools show you what a file *imports*. `onboard impact` performs a **Reverse-Dependency Traversal** (traversing the reverse AST call-graph) to show you exactly what imports *your* file.

Before you commit a refactor in your core module, you need to know exactly which external microservices will fail. This tool maps the exact architectural blast radius of a specific function signature, highlighting vulnerable downstream consumers right in your terminal before your CI/CD pipeline even boots up.

## Usage

```bash
onboard impact --branch <branch_name>
# or
onboard impact --target <file_path>
```

## How It Works

`onboard impact` leverages the internal SQLite graph cache built by the CLI. It takes your target file or current git branch diff, runs a reverse-lookup query on the `edges` table, and recursively maps all nodes that have a directed edge pointing *towards* your target.

It then calculates a "Blast Radius Score":
* **High Score:** The code is a deeply embedded core dependency (like a database connector or an authentication middleware). Exercise extreme caution.
* **Low Score:** The code is likely a leaf node (like a specific API route or an isolated UI component). Safe to modify.

## Example Output

```bash
onboard impact --branch feat/auth-refactor
```

**Terminal Output:**
```plaintext

⚠️  HIGH IMPACT DETECTED ⚠️

Blast Radius: 42 files depend on these modules.
Direct Dependencies:
 - cmd/api/server.go
 - pkg/routes/users.go
 - pkg/routes/orders.go
 - pkg/routes/payments.go
... (and 38 more)

Recommendation: Exercise extreme caution. Changes require extensive integration testing.
```
