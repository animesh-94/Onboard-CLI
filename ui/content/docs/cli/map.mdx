Stop relying on messy whiteboard diagrams that are out of date the minute they are drawn. 

The **Visual Mapper** (`onboard map`) reads your local SQLite database of AST dependencies and generates a stunning, interactive dependency graph. Because it runs locally, you can visualize enterprise-scale microservices without any source code ever leaving your machine (zero telemetry, air-gapped ready).

## Usage

```bash
onboard map --serve --port 3000
```

## How It Works

1. **Local Node Resolution:** It queries the local SQLite `nodes` and `edges` tables generated by the `init` command.
2. **Layout Engine:** It applies a physics-based force-directed graph algorithm to position clusters of related code (e.g. grouped by package or module).
3. **Interactive Canvas:** It serves a local React SPA with a canvas (like the one you see on the homepage) where you can pan, zoom, drag nodes, and instantly see the architecture of your system.

## Exporting for Third-Party Tools

If you don't want to use the built-in React visualizer, Onboard CLI's open architecture allows you to export the graph to standard JSON formats, meaning no vendor lock-in. 

```bash
onboard map --export graph.json
```

You can then import this `graph.json` into tools like Gephi, D3.js, or Cytoscape.
