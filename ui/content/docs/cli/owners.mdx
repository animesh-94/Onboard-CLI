Bypass superficial `git blame` logs. The **Context Owner Engine** cross-references deep commit history with AST node density to find the true architect of any codebase segment. 

Whether you are looking at a localized memory allocator or a distributed scheduling service, `onboard owners` cuts through legacy noise to instantly identify the engineer who actually understands the active business logic.

## Usage

```bash
onboard owners --target <file_path_or_directory>
```

## How It Works

Traditional blame features tell you who last touched a line of code (often someone who just ran an auto-formatter or updated an import). 

Onboard CLI solves this by reading the Abstract Syntax Tree (AST) to measure the **cognitive weight** of the code, and then tying it to git history:

1. **AST Density Mapping:** It assigns weight to code blocks based on logic complexity (e.g. nested loops, complex conditionals, and heavy algorithms get more weight than simple variable declarations).
2. **Git History Correlation:** It correlates these weighted blocks back through the git tree to find the original authors of the core logic.
3. **Primary Architect Identification:** It aggregates the results and tells you exactly who the Primary context owner is, saving you hours of guessing who to tag for a PR review.
