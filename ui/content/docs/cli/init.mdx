Before Onboard-CLI can analyze, visualize, or lint your architecture, it must build a local intelligence graph. The `init` command handles the scaffolding of the `.onboard` directory and sets up the base `onboard.yaml` configuration file for your project.

## Usage

```bash
onboard init
```

## What happens under the hood?

When you run `onboard init`, the CLI performs the following actions:

1. **Scaffolds Directories**: It creates a hidden `.onboard` directory in the root of your project. This directory will act as the home for your local SQLite graph cache, topology dumps, and rulesets.
2. **Generates Config**: It drops a base `onboard.yaml` file into your project. This file is where you can configure specific settings for the CLI (e.g., ignoring specific legacy directories, configuring custom framework extensions, etc.).
3. **Prepares for Mapping**: The `.onboard/cache.db` file is provisioned and the SQLite tables and indexes are initialized, preparing the engine for the `map`, `drift`, and `export` commands.
